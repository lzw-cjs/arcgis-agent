**Implementation:**

Replace the existing `__init__` method (lines 11-13):

```python
def __init__(self):
    import arcpy  # LAZY: inside constructor, not at module level
    self._arcpy = arcpy

    # Check out required license extensions (ROADMAP: CheckOutExtension + try/finally)
    self._checked_out_extensions = []
    for ext_name in ["spatial"]:
        try:
            status = self._arcpy.CheckExtension(ext_name)
            if status not in ("Available", "CheckedOut"):
                continue
            if status == "CheckedOut":
                self._checked_out_extensions.append(ext_name)
                continue
            self._arcpy.CheckOutExtension(ext_name)
            self._checked_out_extensions.append(ext_name)
        except Exception:
            pass  # Non-fatal: individual tool calls will fail with clear arcpy errors if extension is truly needed
```

**Key design decisions:**
1. Only "spatial" extension is checked out — the most commonly needed extension for geoprocessing tools (Spatial Analyst)
2. Extension already checked out by another process (`"CheckedOut"`) is recorded but not re-checked-out
3. `"Unavailable"` status (license not present) is silently skipped — individual tool calls will fail with clear ArcPy errors
4. `try/except` around the entire checkout block ensures no crash on any unexpected arcpy error during init
5. `self._checked_out_extensions` tracks successfully checked-out extensions for inspection/debugging
