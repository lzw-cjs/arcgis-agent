**2. `src/arcgis_agent/models/result.py`** -- Pydantic Result model.

```python
from pydantic import BaseModel, Field
from typing import Any, Optional

class Result(BaseModel):
    success: bool
    code: str = "OK"
    message: str = ""
    data: Optional[dict[str, Any]] = None
    warnings: list[str] = Field(default_factory=list)

    @classmethod
    def ok(cls, data=None, message="OK"):
        return cls(success=True, code="OK", message=message, data=data)

    @classmethod
    def error(cls, code, message, data=None):
        return cls(success=False, code=code, message=message, data=data)

    @classmethod
    def from_exception(cls, exc):
        from arcgis_agent.exceptions import map_exception_to_code
        code = map_exception_to_code(exc)
        return cls(success=False, code=code, message=str(exc))

    def to_json(self, indent=2):
        return self.model_dump_json(indent=indent)
```

Key rules:
- Use `model_dump_json()` NOT `model_dump()` + `json.dumps()` (Pitfall 5 from RESEARCH.md)
- `warnings` uses `Field(default_factory=list)` to avoid mutable default
- `from_exception()` imports `map_exception_to_code` inside method to avoid circular import
