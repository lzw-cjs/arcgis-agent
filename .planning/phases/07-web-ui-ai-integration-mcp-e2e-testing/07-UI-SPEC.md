|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 400 | 1.4 |
| Heading | 16px | 600 | 1.3 |

**Constraints:**
- Maximum 2 font weights in use: 400 (regular) and 600 (semibold).
- No font weight 700 or 800 permitted.
- All text must pass WCAG AA contrast ratio (4.5:1 for body text).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF | Page background, chat panel surface |
| Secondary (30%) | #F5F5F5 | Sidebar, message bubbles (incoming), card backgrounds, input backgrounds |
| Accent (10%) | #1677FF | Primary buttons, links, active states, selected items, send button, focus rings |
| Destructive | #FF4D4F | Delete buttons, error text, destructive action confirmations only |

**Accent reserved for:** Primary button (发送), text links, active navigation item indicator, selected state highlight, focus ring outline, tool call status indicator (success), SSE event accent dot. Never used for decorative elements, non-interactive text, or background fills.

**Destructive reserved for:** Delete confirmation button, error toast icon, error state text, delete action warning banner. Never used for informational messages, warnings, or non-critical UI.

**Color mode:** Light mode only. No dark mode support in this phase.

**Source:** Ant Design 6 default blue (#1677FF) and red (#FF4D4F) token values. Matches Ant Design ConfigProvider theme token defaults.

| Component | Description |
|-----------|-------------|
| `ChatPanel` | Main chat interface: message list + input box. Uses Ant Design Layout.Content. |
| `MessageBubble` | Single chat message (user or AI). User bubble uses accent (#1677FF) background with white text. AI bubble uses secondary (#F5F5F5) background with dark text. Supports Markdown rendering. |
| `MapPanel` | ArcGIS embedded map in a collapsible panel. Uses Ant Design Collapse. Toggled via a button in the chat header. |

| `InputBox` | Chat input with TextArea, send button (发送), and optional suggestion chips. Character count not enforced. |
| `SuggestionBar` | Horizontal row of preset suggestion buttons below the chat input. Renders Ant Design Button with type="default" and size="small". |

### States Per Component

| Component | States Covered |
|-----------|---------------|
| `MessageBubble` | default (text), tool-call (embedded ToolCallCard), loading (Spin), error (Alert) |
| `ToolCallCard` | running (Spin + blue Tag), success (green Tag), error (red Tag + error message) |
| `InputBox` | default (empty), active (focused, blue border), disabled (during loading) |
| `MapPanel` | collapsed (default), expanded, loading (map tiles fetching) |
| `SuggestionBar` | visible (after tool execution), hidden (default, during loading) |
| `Empty state` | no messages (Empty + heading + body), history loaded (scrollable message list) |

## Interaction Rules

1. **Send flow:** User types in InputBox (TextArea), presses Enter or clicks "发送" button. TextArea clears immediately. MessageBubble (user) appears. AI MessageBubble appears with Spin loading. SSE tokens stream in and render progressively. On tool_call event, ToolCallCard appears inside AI bubble with running state. On tool_end, ToolCallCard updates to success/error. On done event, Spin replaced with final rendered content.

2. **Input disabled during loading:** When loading is true (AI response in progress), InputBox is disabled and send button shows loading Spin. User cannot send another message until current response completes.

3. **Map panel toggle:** Collapsed by default (D-19). Toggle button in chat header opens/closes the panel. When AI response includes a map reference (e.g., "I've updated the map"), a "查看地图" button appears in the message bubble -- clicking it expands the MapPanel.

4. **Suggestion chips:** After a tool execution completes successfully, related suggestion chips appear below the AI message bubble (not in the input area). These are based on the template rules from D-27. Clicking a suggestion sends it as the next user message automatically.

5. **Destructive confirmation:** Modal dialog with two buttons. Primary button is destructive (red, labeled "确定删除"). Secondary button is default (labeled "取消"). Modal must display the item name and the phrase "此操作不可撤销" / "This action cannot be undone."

6. **Error recovery:** Connection errors show a persistent Alert banner at the top with a "重试" (Retry) button. Tool execution errors appear inline in the ToolCallCard with the error message and a "重试" button that resends the last user message.

7. **Scroll behavior:** Message list auto-scrolls to bottom when a new message or SSE chunk arrives, unless the user has manually scrolled up to read history. If user is scrolled up, a "向下滚动" (Scroll to bottom) floating button appears.

8. **Keyboard shortcuts:**
   - Enter: Send message (when InputBox is focused)
   - Shift+Enter: New line in InputBox
   - Escape: Close expanded MapPanel, close Modal

## Accessibility Minimum

- All form inputs have associated labels (visually hidden if needed, but present in DOM)
- Focus indicators visible on all interactive elements (accent #1677FF 2px outline, Ant Design default)
- Tab order follows visual layout: InputBox -> Send button -> Map toggle -> Message list (roving tabindex)
- Color is never the sole differentiator for status -- tool call success/error always includes a text label alongside the colored Tag
- Modal focus is trapped within the Modal when open; focus returns to trigger button on close
- All images (if any) have alt text
- `lang="zh-CN"` on `<html>` element, with individual English messages wrapped in `lang="en"` spans if mixed
