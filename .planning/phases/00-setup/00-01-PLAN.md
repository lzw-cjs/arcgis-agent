激活 arcgis-agent 环境后，安装 mcp、rich、hatchling。

**偏离 CONTEXT.md `--no-deps` 决策的说明：** CONTEXT.md 建议使用 `pip install --no-deps`，但 RESEARCH.md 验证确认 mcp 1.27.1 的依赖链（anyio, httpx, jsonschema, pydantic）与 Esri 锁定的 numpy 1.24.3 / pandas 2.0.2 无交集，普通 `pip install` 安全。如果出问题再回退到 `--no-deps`。

```bash
conda activate arcgis-agent
pip install mcp rich hatchling
```

安装完成后逐个验证：

```bash
python -c "from mcp.server.fastmcp import FastMCP; print('mcp OK')"
python -c "from rich.console import Console; print('rich OK')"
python -c "import hatchling; print('hatchling OK')"
python -c "import pydantic; print('pydantic', pydantic.__version__)"
python -c "import arcpy; print('arcpy OK')"
```

关键验证点：
- mcp 可导入（FastMCP 可用）
- pydantic 版本 >= 2.11.0（mcp 依赖要求）
- arcpy 未被破坏

如果 `pip install mcp` 导致 numpy/pandas 被升级破坏 arcpy，回退到 `pip install --no-deps m` 然后手动安装 mcp 的依赖：
```bash
pip install --no-deps mcp
pip install anyio httpx httpx-sse jsonschema pydantic-settings
```
